# PRD

# AI 기반 채용공고 맞춤 검색 시스템 PRD

## 1. 배경 (Background)

현재 사람인 등 채용 포털에서 제공되는 검색 기능은 키워드 기반으로 제한적이다. 사용자는 “AI 엔지니어 신입 채용”, “서울 근처 하이브리드 근무”, “연봉 5천 이상 데이터 분석가” 등 자연어로 다양한 조건을 복합적으로 표현한다. 그러나 기존 검색은 단순 키워드 매칭이라서 사용자가 원하는 채용공고를 정확히 찾기 어렵다.

따라서 AI를 활용해 자연어 질의를 이해하고 의미 기반으로 채용공고를 검색·추천하는 시스템이 필요하다.

## 2. 목표 (Goals)

- 사용자의 자연어 질문을 이해하고 적합한 채용공고를 반환한다.
- 기존 키워드 검색 대비 더 높은 정확도와 사용자 만족도를 달성한다.
- 검색된 공고에서 원하는 조건(지역, 경력, 연봉, 근무형태 등)을 추출 및 필터링 가능하게 한다.
- 서비스 반응 속도는 실시간 검색 경험을 유지할 수 있도록 최적화한다.
- 부족한 정보가 있을 경우 사용자에게 추가 질의를 요청하여 검색 정확도를 높인다.

## 3. 주요 기능 (Key Features)

### 3.1 사용자 입력 처리

- 자연어 질의 입력 (예: “서울에서 2년차 이상 AI 엔지니어”)
- 자동 파싱: 질의에서 직무, 지역, 경력, 급여, 고용형태 등의 조건 추출
- 부족한 정보가 있는 경우 LLM이 사용자에게 추가 질문하여 필요한 조건 수집

### 3.2 채용공고 데이터 수집 및 전처리

- 사람인 채용공고 HTML 크롤링 및 파싱
- 주요 필드 추출: 회사명, 직무, 경력, 연봉, 근무지역, 근무형태, 복리후생, 접수기간 등
- 벡터 임베딩 생성 (예: Sentence-BERT, OpenAI Embeddings 등)

### 3.3 검색 및 추천

- 의미 기반 검색 (Semantic Search): 사용자 질의 임베딩과 공고 임베딩 간 유사도 계산
- 필터링 로직: 질의 내 조건(지역, 경력, 연봉 등)을 기반으로 추가 필터 적용
- 랭킹 알고리즘: 관련도 점수 + 필터 조건 충족도 기반으로 정렬

### 3.4 결과 제공

- 상위 N개의 채용공고 추천
- 결과에는 공고 주요 정보(회사명, 직무명, 경력/연봉 조건, 링크 등) 포함
- 상세 정보는 클릭 시 원문 채용공고 페이지 연결

## 4. 비기능 요구사항 (Non-functional Requirements)

- 성능: 검색 응답 시간 2초 이내
- 확장성: 새로운 데이터셋 및 직군 확장 가능
- 정확도: 키워드 검색 대비 Top-10 정확도 최소 20% 개선

## 5. 사용자 시나리오 (User Scenarios)

### 시나리오 1 – 신입 취업 준비생

- 입력: “서울에서 AI 관련 신입 채용 공고”
- 처리: 질의에서 [지역=서울], [직무=AI], [경력=신입] 추출
- 출력: 서울 지역에서 AI 신입을 채용하는 공고 리스트
- 부족한 정보 감지: 근무형태가 명시되지 않은 경우, LLM이 사용자에게 “선호하는 근무형태가 있나요?” 질문

### 시나리오 2 – 경력직 이직 희망자

- 입력: “데이터 분석가 3년 이상 연봉 5천 이상”
- 처리: [직무=데이터 분석], [경력 ≥ 3년], [연봉 ≥ 5000]
- 출력: 조건에 맞는 경력직 채용공고 추천
- 부족한 정보 감지: 지역이 명시되지 않은 경우, LLM이 사용자에게 “근무 희망 지역을 알려주세요” 질문

## 6. 성공 지표 (Success Metrics)

- 정확도 지표: Hit@k, Precision@k, Recall@k
- 사용자 만족도: 추천 결과 만족도 설문 ≥ 80%
- 사용자 상호작용: 부족한 정보 질문 후 답변 완료율 ≥ 90%

## 7. 기술 스택 (Tech Stack)

- 크롤링/파싱: Selenium, BeautifulSoup
- 백엔드: FastAPI
- AI/NLP: FAISS/HNSW, Hugging Face Transformers

## 9. 리스크 및 고려사항 (Risks & Considerations)

- 사람인 웹사이트 구조 변경 시 크롤링 로직 수정 필요
- 임베딩 모델 성능에 따라 검색 품질 변동 가능
- 실시간성 확보를 위한 인덱스 업데이트 전략 필요
- 부족한 정보 감지 및 추가 질의 로직 구현 난이도 고려

## 10. 전체 프로세스 흐름 (Process Flow)

1. 사용자 질문 입력
2. LLM : URL 쿼리 맵을 사용하여 URL 생성
3. 부족한 정보 감지 및 사용자에게 추가 질문
4. Selenium 크롤링
5. BeautifulSoup 주요 내용 파싱
6. 임베딩 모델 처리
7. 리트리버 검색
8. LLM 최종 응답 생성
