# OpenAI NER 50건 평가 분석

## 요약

- 50건 기준 OpenAI JSON-schema NER의 전체 슬롯 추출 성능은 양호하다.
- Micro F1은 `0.921409`, Macro F1은 `0.909859`로 확인됐다.
- 핵심 병목은 `지역` 슬롯이며, `직무`, `경력`, `학력`은 비교적 안정적이다.

## 수치 해석

- `직무` 정확도 `94.00%`, `경력` 정확도 `95.92%`, `학력` 정확도 `98.00%`:
  사용자가 직무, 경력, 학력을 문장에 명시적으로 포함한 경우에는 대부분 정확히 추출했다.
- `지역` 정확도 `54.00%`:
  전체 성능 저하의 대부분이 여기서 발생한다. 문장 단위 Exact Match가 `53.06%`까지 떨어진 것도 지역 슬롯 오차 영향이 크다.
- Micro precision `1.000000`, recall `0.854271`:
  잘못된 값을 과하게 예측하기보다는, 필요한 값을 덜 잡거나 정규화 단계에서 일치하지 않는 형태로 반환하는 경향이 강하다.

## 주요 실패 패턴

- 실패의 중심은 `지역` 추출 및 정규화 불일치다.
- 사용자 원문에는 `강남`, `성남`, `용산구`, `서초구`, `근처`, `쪽` 같은 세부 지역 표현이 자주 포함된다.
- 모델은 `서울 강남`, `경기 성남`, `서울 용산구`처럼 더 구체적인 표면형을 반환하는 반면, 평가 gold는 `서울`, `경기` 같은 상위 canonical 값으로 비교된다.
- 즉, 의도 이해 자체는 맞더라도 canonical 기준 비교에서는 오답으로 처리되는 경우가 많다.

## 해석상 주의점

- 현재 지표는 “의미 이해”보다 “정규화 후 canonical 값 일치”에 더 민감하다.
- 따라서 `지역` 슬롯 수치는 모델이 지역을 전혀 이해하지 못해서라기보다, 세부 지역 표현을 상위 지역 값으로 접지하지 못하는 문제를 크게 반영한다.
- 반대로 `직무`, `경력`, `학력`은 canonical 기준에서도 높은 정확도를 보여, 현재 OpenAI NER 경로가 이 세 슬롯에는 충분히 실용적이라는 해석이 가능하다.
- 실패 호출이 `1/50` 존재하므로, 운영 문서에는 성능 수치와 함께 호출 안정성 이슈도 같이 적는 편이 맞다.

## 결론

- 현 시점 OpenAI NER 경로는 `직무`, `경력`, `학력` 추출 품질은 좋다.
- 서비스 관점에서 가장 먼저 보완해야 할 항목은 `지역` 정규화 정책이다.
- 특히 구/시/근처/쪽 같은 표현을 `서울` 또는 `경기`로 안정적으로 매핑하는 규칙을 정리하면 Sentence Exact Match와 Recall이 함께 개선될 가능성이 높다.
