파이썬 코드 작성 및 수정 작업에서는 `python-simple-rule` 스킬을 우선 사용한다.
커밋 메시지 작성 및 수정 작업에서는 `commit-message-rule` 스킬을 우선 사용한다.
렌더 배포를 한다면 `render-deploy` 스킬을 사용한다.
openai 문서를 참고해야 한다면 `openai-docs` 스킬을 사용한다.

사용자가 명시적으로 다르게 요청한 경우만 예외로 한다.

## 전체 액세스 권한 사용 시 주의사항

- 사용자의 명시적 요청이 없는 파괴적 명령(`rm -rf`, `git reset --hard`, `git checkout --`, 대량 삭제/덮어쓰기)은 실행하지 않는다.
- 프로젝트 루트 바깥 경로(홈 디렉터리 전체, 시스템 경로 등)는 사용자가 분명히 지정한 경우에만 읽거나 수정한다.
- 운영/배포 환경에 영향을 주는 작업(배포, 인프라 설정 변경, DB 마이그레이션, 외부 서비스 연동)은 실행 전 반드시 사용자 확인을 받는다.
- 비밀정보(`.env`, API 키, 토큰, 개인키, 인증서)는 출력/로그/커밋/외부 전송하지 않는다.
- `git push --force`, 히스토리 재작성, 브랜치/태그 삭제는 사용자가 명시적으로 지시한 경우에만 수행한다.
- 대규모 치환/자동 수정은 먼저 대상 범위와 변경 미리보기(diff)를 확인한 뒤 적용한다.
- 의도가 불명확하거나 위험도가 높은 명령은 추측 실행하지 말고, 짧게 확인 질문 후 진행한다.
- 작업 후에는 변경 파일, 영향 범위, 되돌리는 방법을 함께 공유한다.
